In-game display names are automatically derived and formatted from filename IDs. Underscores become spaces and words are capitalized.

Automatically creates shiny item variants in your creative tab when a corresponding _shiny.png texture file is detected.

🛠️

🚀 Installation & Setup

  1. Install Dependencies

    Ensure you have Fabric Loader and Fabric API for Minecraft 1.21.1.

  2. Add GeckoLib Engine

    Download and install GeckoLib 4.6+ into your mods directory.

  3. Install Mod JAR

    Place the custom-pokedolls-1.0.0.jar file into your `.minecraft/mods/` directory.

  4. Launch & Generate

    Launch the game once. The mod will automatically generate a directory named custom_pokedolls/ in your game root.

📁 Adding Custom Statues

Place your GeckoLib assets into the custom_pokedolls/ directory inside your game root. You can use subdirectories or place files in the root folder.

Recommended Directory Structure 
.minecraft/
└── custom_pokedolls/
    ├── geo/
    │   ├── my_statue.geo.json
    │   └── charizard_doll.geo.json
    └── textures/
        └── block/
            ├── my_statue.png
            ├── my_statue_shiny.png
            └── charizard_doll.png

Naming Conventions & Display Names

  • Base Models: Place your .geo.json file in geo/ and matching .png texture in textures/block/.
  • Automatic Formatting: The mod formats the file ID into a clean title:
    my_statue.geo.json ➔ "My Statue"
    charizard_doll.geo.json ➔ "Charizard Doll"
  • Shiny Variants: Add a second texture file with the _shiny.png suffix (e.g. my_statue_shiny.png). The mod registers a Shiny item with the "Shiny " prefix!

⚡ In-Game Commands

Requires Operator (OP) permission level 2 or singleplayer cheat permissions:

Reload Command
/pokedolls reload

Alias: /custom_pokedolls reload

  • Scans the external custom_pokedolls/ directory for new geometry and textures.
  • Registers new block and item entries dynamically without restarting.
  • Updates existing model properties and synchronizes the runtime resource pack with all connected multiplayer clients.

⚙️ Technical Details & Compatibility

Virtual Resource Pack

The mod operates a runtime built-in resource pack (Custom Pokedolls Virtual Pack) that maps the external directory into Minecraft's resource engine under the custom_pokedolls namespace.

Scale Normalization

Bounding box dimensions are automatically parsed from loaded `.geo.json` files to compute a normalized rendering scale, ensuring statues display proportionally across inventories and worlds.

Multiplayer Synchronization

When installed on a dedicated server, executing `/pokedolls reload` automatically dispatches network synchronization packets (`ReloadResourcesPayload`) to all connected clients.

📖 Official Server & Gameplay Documentation

🎮 Gameplay & Mechanics Overview

Who's That Pokémon? (WTP) allows players to trigger interactive guessing sessions where they are presented with a mysterious Pokémon silhouette in a custom GUI screen. Guessing correctly within the time limit awards leaderboard points and configurable items or custom loot table drops.

🌑

Silhouette GUI Screen

A custom anime-inspired GUI screen opens displaying a darkened silhouette of a random Pokémon from your configured list.

🌐

Flexible Name Recognition

Accepts standard English Pokémon names (e.g. pikachu, charizard, squirtle) in chat or GUI.

🏆

Leaderboard System

Tracks player scores persistently in wtp_leaderboard.json. View server top champions anytime using /wtp top.

🎁

Loot Table Rewards

Rewards winners via vanilla/modpack loot tables (factoring in player Luck) or fallback item lists with score bonus scaling!

⏳

Cooldowns & Penalties

Includes personal cooldowns (default: 300s), attempts limits (default: 3), and score deductions for timeouts or wrong guesses.

💡

Hint System

Optional hint mechanism reveals the first letter and total character length (e.g. P (7)) for a minor score deduction.

💬 Complete Command Reference

All commands use the root /wtp prefix. Admin commands require OP level 2+ or LuckPerms permissions:

Command Description
/wtp (or /wtp play) Opens the interactive Who's That Pokémon silhouette GUI screen.
/wtp top Displays the Top 10 leaderboard with ranks, names, and scores.
/wtp reload Reloads configuration file and verifies server status (OP 2+).
/wtp reset [player] Resets personal or target guessing cooldown immediately (OP 2+).

⚙️ Difficulty & Scoring

Configure dynamic difficulty tiers and leaderboard score mechanics in whosthatpokemon-server.toml:

  • Easy Tier: Silhouette has higher transparency and players get +15 extra seconds on the timer.
  • Medium Tier: Standard black silhouette rendering and default 30-second timer.
  • Hard Tier: Zoomed-in/cropped silhouette to challenge veterans, with -10 seconds less time!
  • Scoring Formulas:
    Correct Guess: +50 Points & Reward Drops
    Wrong Guess / Timeout: -10 Points
    Taking a Hint: -15 Points (Reveals Letter & Length)

📝 Server Configuration (whosthatpokemon-server.toml)

When started, the mod generates a clean TOML configuration file allowing full customization of cooldowns, allowed Pokémon lists, fallback item rewards, and rate limiting:

serverconfig/whosthatpokemon-server.toml (snippet) 
[general]
# Cooldown time in seconds before a player can play again (default: 300 / 5 min)
cooldown_seconds = 300

# Format: 'internal_name' or 'internal_name=display_name' e.g. 'pikachu'
allowed_pokemon = [
    "pikachu",
    "bulbasaur",
    "charmander",
    "squirtle"
]

# Fallback rewards table if loot_table_id is not used
rewards = ["minecraft:diamond;1", "minecraft:emerald;5"]
reward_points = 50
penalty_wrong_guess = 10
loot_table_id = "" # Optional: e.g. 'minecraft:chests/simple_dungeon'
max_attempts = 3
session_timeout_seconds = 30
difficulty = "medium"
