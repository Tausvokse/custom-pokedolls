Transform Your World with Dynamic 3D Statues

Load custom Pokedolls directly from your file system using GeckoLib models and textures without ever rebuilding your mod or restarting Minecraft.

1.21.1 Minecraft Fabric
GeckoLib 3D Engine
Live Hot Reloading

✨ Key Features

📂

Dynamic Asset Loading

Load custom 3D .geo.json models and .png textures from an external folder without modifying the mod JAR or restarting.

🏷️

Automatic Item Naming

In-game display names are automatically derived and formatted from filename IDs. Underscores become spaces and words are capitalized.

🌟

Optional Shiny Variants

Automatically creates shiny item variants in your creative tab when a corresponding _shiny.png texture file is detected.

🔄

Live In-Game Reloading

Use `/pokedolls reload` to instantly scan your folders, register new statues, and sync virtual resource packs on the fly.

🛠️

Interactive GUI Chisel

Right-click any statue with the Pokedoll Chisel to adjust rotation, scale, position offsets, animation poses, and glowing luminescence.

🛡️

Crash Protection

Built-in fallback rendering prevents game crashes or corruption if a model or texture is accidentally deleted while placed.

🚀 Installation & Setup

  1. Install Dependencies

    Ensure you have Fabric Loader and Fabric API for Minecraft 1.21.1.

  2. Add GeckoLib Engine

    Download and install GeckoLib 4.6+ into your mods directory.

  3. Install Mod JAR

    Place the custom-pokedolls-1.0.0.jar file into your `.minecraft/mods/` directory.

  4. Launch & Generate

    Launch the game once. The mod will automatically generate a directory named custom_pokedolls/ in your game root.

📁 Adding Custom Statues

Place your GeckoLib assets into the custom_pokedolls/ directory inside your game root. You can use subdirectories or place files in the root folder.

Recommended Directory Structure 
.minecraft/
└── custom_pokedolls/
    ├── geo/
    │   ├── my_statue.geo.json
    │   └── charizard_doll.geo.json
    └── textures/
        └── block/
            ├── my_statue.png
            ├── my_statue_shiny.png
            └── charizard_doll.png

Naming Conventions & Display Names

  • Base Models: Place your .geo.json file in geo/ and matching .png texture in textures/block/.
  • Automatic Formatting: The mod formats the file ID into a clean title:
    my_statue.geo.json ➔ "My Statue"
    charizard_doll.geo.json ➔ "Charizard Doll"
  • Shiny Variants: Add a second texture file with the _shiny.png suffix (e.g. my_statue_shiny.png). The mod registers a Shiny item with the "Shiny " prefix!

⚡ In-Game Commands

Requires Operator (OP) permission level 2 or singleplayer cheat permissions:

Reload Command
/pokedolls reload

Alias: /custom_pokedolls reload

  • Scans the external custom_pokedolls/ directory for new geometry and textures.
  • Registers new block and item entries dynamically without restarting.
  • Updates existing model properties and synchronizes the runtime resource pack with all connected multiplayer clients.

🔨 Using the Pokedoll Chisel

The Pokedoll Chisel is an interactive in-game tool located in the Custom Pokedolls creative tab.

1

Place any custom Pokedoll statue block in the world.

2

Hold the Pokedoll Chisel in your main hand and Right-Click the statue.

3

The Pokedoll Customization Screen GUI will open with advanced controls:

Position Offsets: Fine-tune X, Y, and Z rendering positions.
Rotation Slider: Smooth 360° rotation slider.
Glowing Effect: Toggle luminescence (Light Level 15).
Auto-Rotate: Continuous spinning showcase animation.
Pose Selection: Switch between GeckoLib animation poses.
Reset Controls: Instant reset to default scale and rotation.

⚙️ Technical Details & Compatibility

Virtual Resource Pack

The mod operates a runtime built-in resource pack (Custom Pokedolls Virtual Pack) that maps the external directory into Minecraft's resource engine under the custom_pokedolls namespace.

Scale Normalization

Bounding box dimensions are automatically parsed from loaded `.geo.json` files to compute a normalized rendering scale, ensuring statues display proportionally across inventories and worlds.

Multiplayer Synchronization

When installed on a dedicated server, executing `/pokedolls reload` automatically dispatches network synchronization packets (`ReloadResourcesPayload`) to all connected clients.
